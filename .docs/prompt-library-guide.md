# 提示词中控室 - 使用指南

## 🎯 功能概述

提示词中控室是一个本地化的提示词管理系统，帮助您高效组织和使用各种 AI 提示词。

### 核心功能

1. **分类管理**：支持多级分类（如：`AI助手/代码生成/Python`）
2. **全文搜索**：使用 Fuse.js 实现模糊搜索，支持标题、内容、标签搜索
3. **一键复制**：点击复制按钮即可将提示词复制到剪贴板
4. **使用统计**：自动记录每个提示词的使用次数
5. **导入导出**：支持 JSON 格式的批量导入导出
6. **标签系统**：为提示词添加多个标签，方便分类和搜索

## 🚀 快速开始

### 1. 打开提示词库

在 Timer 主界面的左侧边栏，点击 **书本图标**（📖）即可打开提示词库窗口。

### 2. 创建第一个提示词

1. 点击右上角的 **"新建"** 按钮
2. 填写提示词信息：
   - **标题**：简短描述，如 "Python 代码生成助手"
   - **分类**：用 `/` 分隔的多级分类，如 "AI助手/代码生成/Python"
   - **标签**：用逗号分隔，如 "python, 代码, 开发"
   - **内容**：完整的提示词文本
3. 点击 **"保存"**

### 3. 搜索提示词

在顶部搜索框输入关键词，系统会自动搜索：
- 提示词标题
- 提示词内容
- 分类名称
- 标签

支持模糊匹配，例如输入 "pythn" 也能匹配到 "python"。

### 4. 使用提示词

1. 找到需要的提示词
2. 点击 **复制图标**（📋）
3. 提示词内容已复制到剪贴板
4. 粘贴到任何需要的地方（如 AI 对话框）

## 📂 分类管理

### 分类命名规范

使用 `/` 分隔多级分类，例如：

```
AI助手/代码生成/Python
AI助手/代码生成/JavaScript
AI助手/文案优化
数据分析/SQL
数据分析/数据可视化
```

### 分类视图

点击右上角的 **树形图标**（🌳）切换到分类视图，可以：
- 查看所有分类及其提示词数量
- 点击分类名称筛选该分类下的所有提示词
- 点击 "全部" 返回所有提示词

---

## 🏷️ 标签系统

### 添加标签

在创建或编辑提示词时，在 "标签" 字段输入多个标签，用逗号分隔：

```
python, 代码, 开发, 后端
```

### 标签搜索

在搜索框输入标签名称（带或不带 `#`），即可搜索包含该标签的所有提示词。

---

## 📊 使用统计

每次点击复制按钮时，系统会自动：
1. 增加该提示词的使用次数
2. 更新 "最后使用时间"

在提示词卡片上会显示使用次数（如："使用 15 次"）。

---

## 💾 导入导出

### 导出提示词

1. 点击右上角的 **下载图标**（⬇️）
2. 系统会生成 `prompts-YYYY-MM-DD.json` 文件
3. 保存到本地作为备份

### 导入提示词

1. 点击右上角的 **上传图标**（⬆️）
2. 选择之前导出的 JSON 文件
3. 系统会自动合并导入（避免 ID 冲突）

### 数据格式

导出的 JSON 文件格式：

```json
[
  {
    "id": "prompt-1704723600000-abc123",
    "title": "Python 代码生成助手",
    "content": "你是一个专业的 Python 开发者...",
    "category": "AI助手/代码生成/Python",
    "tags": ["python", "代码", "开发"],
    "createdAt": 1704723600000,
    "updatedAt": 1704723600000,
    "usageCount": 15
  }
]
```

## 💡 使用技巧

### 1. 建立个人提示词库

按照工作场景分类，例如：

```
工作/
├── 代码开发/
│   ├── Python
│   ├── JavaScript
│   └── SQL
├── 文档写作/
│   ├── 技术文档
│   └── 产品文档
└── 数据分析/
    ├── 数据清洗
    └── 可视化

学习/
├── 语言学习/
│   ├── 英语
│   └── 日语
└── 技能提升/
    └── 编程

生活/
├── 健康
└── 旅行
```

### 2. 使用标签增强搜索

为同一提示词添加多个维度的标签：

```
标题：Python 异步编程助手
分类：AI助手/代码生成/Python
标签：python, 异步, async, 并发, 高级
```

这样无论搜索 "异步"、"async" 还是 "并发"，都能找到这个提示词。

### 3. 定期备份

建议每周导出一次提示词库，保存到云盘或 Git 仓库。

### 4. 团队共享

导出提示词库后，可以分享给团队成员，大家导入后即可使用相同的提示词集合。

## 📝 数据存储

所有提示词数据存储在浏览器的 LocalStorage 中：

- **存储位置**：`localStorage['widget-prompts-library-v1']`
- **数据格式**：JSON 数组
- **容量限制**：通常 5-10MB（足够存储数千条提示词）

### 数据安全

- ✅ 数据完全本地化，不会上传到任何服务器
- ✅ 支持导出备份
- ⚠️ 清除浏览器数据会删除所有提示词（请定期备份）

## 🐛 常见问题

### Q1: 提示词丢失了怎么办？

**A**: 如果有备份文件，直接导入即可。如果没有备份，数据无法恢复（LocalStorage 清除后数据永久丢失）。

**建议**：定期导出备份。

### Q2: 可以存储多少条提示词？

**A**: 理论上可以存储数千条。实际限制取决于：
- LocalStorage 容量（通常 5-10MB）
- 每条提示词的长度

对于普通使用（< 1000 条），完全没有问题。

### Q3: 搜索速度会变慢吗？

**A**: 在 1000 条以内，搜索速度 < 10ms，用户感知不到延迟。

### Q4: 可以在多个设备间同步吗？

**A**: 当前版本不支持。可以通过导出/导入手动同步。未来计划添加云端同步功能。

### Q5: 如何批量创建提示词？

**A**: 
1. 手动创建一个提示词模板
2. 导出为 JSON
3. 复制 JSON 格式，批量编辑
4. 导入回系统
